## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

- Use GitHub private vulnerability reporting when available in the target repository.
- If private reporting is unavailable, contact the maintainers through your internal security channel.

When reporting, include:
- A description of the issue and potential impact
- Reproduction steps or proof-of-concept
- Any suggested remediation

We will acknowledge receipt as soon as possible and provide follow-up on triage/remediation.
