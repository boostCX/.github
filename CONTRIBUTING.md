# Contributing

Thanks for contributing.

## How to contribute

1. Create or find an issue for the change.
2. Create a branch from `main` using the naming convention: `feature/<issue-id>-<short-desc>`, `bugfix/<issue-id>-<short-desc>`, or `hotfix/<issue-id>-<short-desc>`.
3. Keep changes focused and scoped.
4. Add or update tests when applicable.
5. Open a pull request targeting `main` using the default PR template.
6. Delete your branch after it is merged.

For the full branching strategy, see the engineering handbook.

## Pull request expectations

- Clear description of what changed and why
- Validation evidence (tests, lint, manual checks)
- Linked issue when applicable

## Code of conduct

By participating, you agree to follow the repository's code of conduct and collaboration standards.
